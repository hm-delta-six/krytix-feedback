# Krytix — Feedback & Feature Requests

> **Krytix** ist ein browser-basierter Multi-Cloud-Dateimanager.
> Dieses Repository dient ausschließlich als Feedback- und Feature-Request-Kanal.
> Der Quellcode ist in einem separaten privaten Repository.

---

## 💬 Feedback geben

### Neuen Cloud-Anbieter anfragen
Du möchtest einen bestimmten Cloud-Dienst unterstützt sehen?
→ [Anbieter anfragen](../../issues/new?labels=provider-request&title=Provider+Request%3A+)

### Bug melden
Etwas funktioniert nicht wie erwartet?
→ [Bug melden](../../issues/new?labels=bug&title=Bug%3A+)

### Feature vorschlagen
Eine Idee für eine neue Funktion?
→ [Feature vorschlagen](../../issues/new?labels=feature-request&title=Feature%3A+)

## 👍 Bestehende Anfragen unterstützen

Schau dir [offene Issues](../../issues) an — wenn jemand bereits dasselbe
angefragt hat, stimme mit einem 👍 ab. Das hilft bei der Priorisierung.

---

## Was hier NICHT hingehört

- Supportanfragen zu deinem Account → per E-Mail direkt an uns
- Sicherheitslücken → **nicht** als öffentliches Issue melden,
  sondern per E-Mail (support@krytix.io)
